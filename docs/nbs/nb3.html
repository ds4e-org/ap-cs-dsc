2.0 - Visualizing Numerical (Quantitative) Data¶

Visualizations help us develop an overall understanding of our data. In this section, we use Matplotlib to create plots and charts that allow us to examine distributions, compare groups, and identify patterns that may not be obvious from numerical summaries alone.

When working with quantitative data, we describe a distribution in terms of its shape (skew and number of peaks), center (a typical value), and variation (spread).

  • Skew describes whether the data is balanced or has a longer tail on one side (left or right).
  • Number of peaks describes how many high points (modes) appear in the distribution.
  • Center is a typical or representative value (such as the mean or median).
  • Spread describes how much the values differ from one another. The range is one way to measure spread and is calculated as the difference between the maximum and minimum values.

Histograms¶

Histograms display how the values of a quantitative variable are grouped across intervals. Here, we use the .hist() method to display a histogram of the variable age.

Instructor Note: gss["age"].hist() uses Pandas’ .hist() method to create a histogram of the age column, while plt.show(); tells Matplotlib to display the plot.
  • Pandas plotting methods (such as .hist()) use Matplotlib internally to draw figures. In contrast, functions that begin with plt. (e.g., plt.xlabel(), plt.ylabel(), plt.title(), plt.show();) come directly from the matplotlib.pyplot module and are used to customize or display the plot.
  • The semicolon at the end of plt.show(); prevents Jupyter from displaying additional, unnecessary output. Without a semicolon, Jupyter displays both the plot and the object returned by the plotting command (e.g., <matplotlib.axes._axes.AxesSubplot at 0x...>.

Boxplots¶

Histograms show how the values of a quantitative variable are distributed across intervals, but they do not explicitly display summary statistics such as the median or quartiles.

Boxplots, on the other hand, summarize the distribution of a quantitative variable by showing the median, quartiles, and potential outliers. Here, the .boxplot() method is used to display a boxplot of wwwhr.

In [9]:
## Run this code but do not edit it. Hit Ctrl+Enter to run the code
# Create a boxplot for wwwhr
# vert=False make he boxplot appear horizontally
gss.boxplot("wwwhr", vert=False, grid=False)

# Label the x-axis and remove the label from the y-axis
plt.xlabel("Hours")
plt.ylabel("")
plt.yticks([])

# Add a title and remove the default subtitle (suptitle)
plt.title("Distribution of Weekly Internet Usage")
plt.suptitle("")
plt.show();
Interpreting the Results:
  • The median (represented by the line inside the box) is around 6–8 hours per week, indicating the typical reported internet usage.
  • The middle 50% of values (represented by the box) falls roughly between 2 and 15 hours per week.
  • The lower end of the distribution is close to 0 hours, suggesting that some respondents report little to no internet use.
  • The upper whisker extends to around 30–40 hours per week, showing the range of most reported values.
  • There are several outliers, with some respondents reporting 75, 90, 100, or even over 160 hours per week.

When working with categorical data, we focus on how observations are distributed across categories. Unlike numerical data, which consists of values that can be measured and used in calculations, categorical data consists of labels or groups (for example, “Pretty Happy” or “Not too happy”). We summarize categorical data using counts or proportions to understand how responses are distributed across categories.

Students may be tempted to compare counts across groups when describing differences in the two-way table. However, because the group sizes differ, counts can be misleading. In this case, students should compare proportions instead, as proportions reflect the share of responses within each group and allow for a more meaningful comparison of distributions across categories.

The two-way table above includes several non-substantive categories, like .i: Inapplicable, .n: No answer, .s: Skipped on Web. These responses to the survey question do not represent reported degree or happiness levels. To focus our comparison on respondents who provided meaningful answers, we need to remove these categories.

In [21]:
## Run this code but do not edit it. Hit Ctrl+Enter to run the code
# Define the degree and happy categories we want to include
deg_cat = ["Less than high school", "High school", "Bachelor's", "Graduate"]
hap_cat = ["Very happy", "Pretty happy", "Not too happy"]

# Create a mask that keeps only rows with selected degree and happy categories
deg_hap_mask = (gss["happy"].isin(hap_cat)) & (gss["degree"].isin(deg_cat))

# Apply the mask to create a filtered dataframe
deg_hap_df = gss[deg_hap_mask].copy()

# Create a two-way frequency table for degree and happiness
pd.crosstab(deg_hap_df["degree"], deg_hap_df["happy"])

Reminder: When comparing groups, do not rely on counts alone. Counts reflect both the size of the group and how responses are distributed, which can lead to misleading conclusions. Instead, compare proportions, as they show the share of responses within each group and provide a more meaningful comparison.

Example: Suppose you are comparing responses to the happy variable across two groups.
